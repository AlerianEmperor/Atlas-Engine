1 # Blue Engine
2 ![Example scene](image.png)
3 ## Introduction
4 This is a cross platform engine that is available on all major platforms including Linux, Windows and MacOS.
5 >**Note:**
6 >MacOS only supports OpenGL up to version 4.1. Some features aren't available there.
7 ## Requirements
8 - OpenGL 4.3 or higher
9 - OpenGL ES 3.2
10 ## Installation
11 The installation is done using CMake. There are two options available: Start a new project with a predefined
12 main file which you can edit. The second option is two use the engine as a subproject in an already existing project.
13 >**Note:**
14 >The engine is only able to compile and run in an 64 bit environment.
15 #### New project using the engine
16 After running CMake you can find the main file at **./src/main.cpp**. Just start your project there, it already
17 contains a main function.
18 #### Excisting project using the engine
19 There exist two options:
20 - You can use the engine as a CMake subproject. Just go ahead and use **add_subdirectory** in the root
21 CMakeLists.txt of your project. Afterwards add **target_link_libraries(YOUR_TARGET ... BlueEngine ${BLUE_ENGINE_LIBS})**. You should be fine.
22 - You can compile the engine and all dependencies as a static library (note that some dependencies also have
23 dynamic libraries). Therefore use the BUILD_LIBRARY option when using CMake. After compiling the library
24 with your favourite build system you shouldn't forget to also copy all the libraries and include them to your project.
25 You can find them in **./lib/YourFavouriteSystem/**.
26 ## Documentation
27 If you want more information have a look into the [Documentation](https://tippex97.github.io/Blue-Engine/index.html)
28 ## General performance tips
29 - Use static/stationary objects and lights as often as possible
30 - Use textures with the same size and format for each material
31 - Set mesh data or any kind of data prior to rendering. More importantly: Don't create data while rendering.
32 - If you are programming additional renderers or content always use the mapping feature of the vertex buffers.
33  Don't use SetData() often, use SetSubData instead.
34  Also try to avoid using OpenGL, use existing engine features instead
35 ## Code Example
36 This code example can also be found in the main file.
37 ```c
38 #include <Engine.h>
39 #include <input/Mouse.h>
40 #include <input/Keyboard.h>
41 
42 int main(int argc, char* argv[]) {
43 
44  Window* window = Engine::Init("../data/shader", "Blue Engine", WINDOWPOSITION_UNDEFINED,
45  WINDOWPOSITION_UNDEFINED, 1280, 720, WINDOW_RESIZABLE | WINDOW_BORDERLESS);
46 
47  Camera* camera = new Camera(47.0f, 2.0f, 1.0f, 200.0f);
48  camera->location = glm::vec3(30.0f, 25.0f, 0.0f);
49  camera->rotation = glm::vec2(-3.14f / 2.0f, 0.0f);
50 
51  Terrain* terrain = new Terrain(9, 7, 16, 0.5f, 30.0f);
52 
53  Texture* texture = new Texture("../data/image.png");
54 
55  window->SetIcon(texture);
56 
57  RenderTarget* target = new RenderTarget(1920, 1080);
58 
59  Scene* scene = new Scene();
60 
61  Cubemap* cubemap = new Cubemap("../data/cubemap/right.png",
62  "../data/cubemap/left.png",
63  "../data/cubemap/top.png",
64  "../data/cubemap/bottom.png",
65  "../data/cubemap/front.png",
66  "../data/cubemap/back.png");
67 
68  scene->sky->skybox = new Skybox(cubemap);
69 
70  scene->postProcessing->chromaticAberration = new ChromaticAberration(0.7f);
71 
72  Mesh* mesh = new Mesh("../data/cube.dae");
73  Mesh* sponzaMesh = new Mesh("../data/sponza/sponza.dae");
74 
75  Actor* actor = new Actor(mesh);
76  Actor* sponza = new Actor(sponzaMesh);
77  sponza->modelMatrix = scale(mat4(1.0f), vec3(0.05f));
78 
79  MasterRenderer* renderer = new MasterRenderer();
80 
81  SceneNode* node = new SceneNode();
82 
83  Light* globalLight = new Light(DIRECTIONAL_LIGHT);
84  globalLight->direction = vec3(0.0f, -1.0f, 0.5f);
85  globalLight->diffuseColor = vec3(253, 194, 109) / 255.0f * 3.0f;
86  globalLight->ambient = 0.05f;
87  globalLight->AddShadow(new Shadow(125.0f, 0.004f, 2048, 3, 0.7f), camera);
88  globalLight->AddVolumetric(new Volumetric(target->width / 2, target->height / 2, 20, -0.5f));
89 
90  node->Add(actor);
91  scene->rootNode->Add(node);
92 
93  scene->Add(sponza);
94 
95  scene->Add(globalLight);
96 
97  node->transformationMatrix = translate(vec3(0.0f, 1.0f, 5.0f));
98 
99  // We create the controller handler
100  MouseHandler* mouseHandler = CreateMouseHandler(camera, 1.5f, 0.25f);
101  mouseHandler->lock = true;
102  KeyboardHandler* keyboardHandler = CreateKeyboardHandler(camera, 7.0f, 0.3f);
103 
104  // For now we will leave the main loop here until we implement a more advanced event system
105  // Our event structure
106  SDL_Event event;
107  bool quit = false;
108 
109  // We need the time passed per frame in the rendering loop
110  unsigned int time = 0;
111 
112  // Handle events and rendering here
113  while (!quit) {
114 
115  unsigned int deltatime = SDL_GetTicks() - time;
116  time = SDL_GetTicks();
117 
118  // Poll all the events
119  while (SDL_PollEvent(&event)) {
120 
121  if (event.type == SDL_QUIT) {
122 
123  // If the SDL event is telling us to quit we should do it
124  quit = true;
125 
126  }
127  else if (event.type == SDL_KEYDOWN && event.key.keysym.sym == SDLK_ESCAPE) {
128 
129  // If the user presses escape we also want to quit
130  quit = true;
131 
132  }
133  else if (event.type == SDL_KEYDOWN && event.key.keysym.sym == SDLK_ESCAPE) {
134 
135  // If the user presses escape we also want to quit
136  quit = true;
137 
138  }
139  else if (event.type == SDL_MOUSEBUTTONDOWN && event.button.button == SDL_BUTTON_LEFT) {
140 
141  mouseHandler->lock = false;
142 
143  }
144  else if (event.type == SDL_MOUSEBUTTONUP && event.button.button == SDL_BUTTON_LEFT) {
145 
146  mouseHandler->lock = true;
147 
148  }
149 
150  }
151 
152  CalculateMouseHandler(mouseHandler, camera, deltatime);
153  CalculateKeyboardHandler(keyboardHandler, camera, deltatime);
154 
155  camera->UpdateView();
156  camera->UpdateProjection();
157 
158  scene->rootNode->transformationMatrix = glm::rotate((float)time / 1000.0f, vec3(0.0f, 1.0f, 0.0f));
159  actor->modelMatrix = glm::rotate((float)time / 500.0f, vec3(0.0f, 1.0f, 0.0f));
160 
161  scene->Update();
162 
163  globalLight->shadow->Update(camera);
164 
165  renderer->RenderScene(window, target, camera, scene);
166 
167  window->Update();
168 
169  }
170 
171  return 0;
172 
173 }
174 ```
175 As you may have seen it's pretty easy to use the engine. One thing I have to warn you is that the rendering parts of the
176 code are **not thread-safe**. So it's recommended to keep the rendering in just one thread. Loading textures, shaders and
177 vertex buffers can be safely done in another thread.
