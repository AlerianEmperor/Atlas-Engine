int32_t  stride,
uint32_t  usage = GL_STATIC_DRAW 
)

Constructs a VertexBuffer object.

Parameters
typeThe type/target of the data, e.g GL_ARRAY_BUFFER
dataTypeThe data type of the data, e.g GL_FLOAT
strideThe number of elements of dataType
usageThe usage of the buffer which is GL_STATIC_DRAW by default.
Note
This is similar to glBufferData. For all available data types see Common Enums. GL_FIXED is not supported.
Remarks
If you want to pass a 4-component vector you would use GL_FLOAT as the data type and 4 as the stride. For a 4x4 matrix data type would also be GL_FLOAT, but stride would equal 16. This is different to all the OpenGL functions, where stride is in bytes.
